# Backend Data Pipeline Scripts

One-time data generation and validation scripts for the campaign planner dashboard.

## 🤖 Automatic Execution

**These scripts run AUTOMATICALLY when the Flask backend starts!**

You don't need to run them manually. The Flask app (`app/initialization.py`) will:
1. Generate inventory on startup
2. Build forecasts on startup
3. Convert to JSON on startup
4. Validate data on startup

See `POST /api/admin/regenerate-data` in backend README to manually trigger regeneration.

## 📁 Location
These scripts are located in `backend/scripts/` and use `../../` relative paths to access project root files.

## 🔧 Available Scripts

### 1. `generate_inventory.py`
Generates inventory data from sales history.

**What it does:**
- Reads `daraz_pak_data.csv` (sales data)
- Finds top 200 SKUs by revenue
- Creates realistic stock levels (2,000-12,000 units per SKU)
- Outputs → `inventory_data.csv` (project root)

**Usage:**
```bash
cd backend/scripts
python generate_inventory.py
```

**Output:**
- `../../inventory_data.csv` (200 SKU inventory records)

---

### 2. `generate_inventory_json.py`
Converts inventory CSV to JSON for React frontend.

**What it does:**
- Reads `inventory_data.csv`
- Converts to JSON format
- Outputs → `frontend/src/inventory_data.json`

**Usage:**
```bash
cd backend/scripts
python generate_inventory_json.py
```

**Output:**
- `../../frontend/src/inventory_data.json` (JSON inventory data)

---

### 3. `build_dashboard.py`
Generates campaign forecasts and dashboard data using Prophet time-series modeling.

**What it does:**
- Reads `daraz_pak_data.csv` (2+ years of sales history)
- Trains Prophet AI model on historical trends
- Forecasts peak dates and GMV for 3 major campaigns:
  - 11-11 (Singles Day): November 11
  - Black Friday: November 24
  - Eid/Ramadan: June 15
- Identifies top 10 hero products per campaign
- Outputs → `frontend/src/dashboard_data.json`

**Usage:**
```bash
cd backend/scripts
python build_dashboard.py
```

**Output:**
- `../../frontend/src/dashboard_data.json` (campaigns, forecasts, hero products)

**Time to run:**
- ~5-10 minutes (Prophet training + forecasting)

**Benefits:**
- Realistic stockout day calculations
- Category metadata from actual sales data
- Seasonal peak detection (automated)

---

### 4. `check_empty_skus.py`
Data quality validation script.

**What it does:**
- Reads `frontend/src/dashboard_data.json`
- Finds hero products with empty/missing SKUs
- Reports validation issues

**Usage:**
```bash
cd backend/scripts
python check_empty_skus.py
```

**Sample output:**
```
bad count 0
```
(Good = count is 0)

---

## 📋 Pipeline Order

Run in this sequence when setting up from scratch:

1. **`generate_inventory.py`** → Creates `inventory_data.csv`
2. **`build_dashboard.py`** → Creates `frontend/src/dashboard_data.json`
3. **`generate_inventory_json.py`** → Creates `frontend/src/inventory_data.json`
4. **`check_empty_skus.py`** → Validates data quality

**Or use the shortcut:**
```bash
cd backend/scripts
python generate_inventory.py && \
python build_dashboard.py && \
python generate_inventory_json.py && \
python check_empty_skus.py
```

## 📦 Dependencies

All scripts require packages from `backend/requirements.txt`:
```bash
pip install -r ../requirements.txt
```

- **pandas** - Data manipulation
- **numpy** - Numerical operations
- **prophet** - Time-series forecasting
- **pystan** - Probabilistic programming (required by Prophet)

This ensures the frontend always has fresh data on each deployment.

## 🐛 Troubleshooting

**"Could not find the CSV file"**
- Make sure you're running from `backend/scripts/` directory
- Verify `daraz_pak_data.csv` exists in project root

**"No module named 'prophet'"**
- Run: `pip install -r ../requirements.txt`

**Prophet takes too long (10+ minutes)**
- Normal for first run on large datasets
- Consider updating `daily_seasonality=False` setting for faster training

**Empty SKUs showing in data**
- Run: `python check_empty_skus.py`
- Scripts auto-filter empty SKUs during processing
